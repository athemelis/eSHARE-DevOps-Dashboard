📊 How is effort calculated? ▼

Effort: Sum of the "Effort" field on each work item (in person-days).

By Type: Effort is shown separately per work item type to avoid double-counting. For example, if a Delivery Slice has 5d effort and its child Tasks also have effort, they are counted separately.

Capacity: Number of team members × working days in the selected period (excludes weekends).

Utilization: Total Effort ÷ Capacity × 100%. Values over 100% indicate over-commitment.

Note: This calculation assumes all team members are available full-time. PTO, holidays, and part-time allocations are not factored in.

📖 How are bug metrics calculated? ▼

Active bugs: Bugs that were active during the selected period (created before period end AND not closed before period start).

Spread effort: Effort is distributed proportionally across each bug's lifetime. E.g., a 15-day bug open for 50 days contributes 15×(7/50)=2.1 days to a 7-day period.

Utilization %: Spread effort ÷ working days in period × 100. Shows what % of an engineer's time was spent on bugs.

Created vs Closed: Bugs created or closed within the selected date range. "Closed" includes both Done (code changes) and Closed (no code changes) states.

📖 How is bug effort tracked? ▼

Estimate (Bug Effort): Initial estimate entered by Bug Owner when bug is created.

Actual (Effort Rollup): Sum of effort from child tasks, entered by engineers as they work.

Divergence: Actual − Estimate. Positive = took longer than estimated, Negative = took less.

Variance: The divergence as a percentage of the estimate.

Data Quality Checks

📖 What makes data "quality"? ▼

Fully Classified Bug: A bug with Bug Type, Feature, and Component all set.

Tasks Missing Effort: Tasks where engineers haven't logged their actual effort (Effort field is empty or 0).

Bugs Without Tasks: Bugs with estimates but no task breakdown (work not yet planned).

Orphan Tasks: Tasks not linked to any parent work item.

Same-Type Parent: Work items whose parent is the same type (e.g., Task under Task) - unusual hierarchy.

Cross-Team Assignment: Work item assigned to someone not in the team (based on Area Path vs Org Chart).

Unknown Assignee: Work item assigned to someone not found in any team in the Org Chart.
